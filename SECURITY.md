# Security Policy

## Supported versions

Only the latest published release is supported with security fixes.

## Reporting a vulnerability

Use GitHub private vulnerability reporting for issues that could delete,
overwrite, disclose, or move data outside an approved recycle-bin boundary.
Do not include exploitable details, credentials, private filenames, or
diagnostic archives in a public issue.

For non-sensitive defects, open a GitHub issue and include the Unraid version,
filesystem layout, affected path class, plugin version, and reproducible steps.
